目标场景和背景说明

核心目标是将风险管理相关业务作为独立应用开发，降低与其他业务系统的耦合度，提升维护与开发效率。


功能说明

【风险管理】用于查看、处置、统计风险记录，具体功能包括：

风险记录管理：支持风险记录的查询、筛选、排序，查看风险详情。

风险处置：对风险记录进行处置，选择处置方式，填写处置说明。

批量操作：支持批量处置风险记录，提高处置效率。


字典说明：选自系统配置->字典管理

1. 风险分类：就是决策分类

2. 风险等级

等级     说明

高风险     可能导致重大损失或系统故障的风险

中风险     可能影响业务正常运行的风险

低风险     影响较小，可安排后续处理的风险

3. 风险状态

状态     说明

待处置     已识别但尚未进行处置的风险

已处置     已完成处置的风险

4. 处置方式

方式     说明

放行     确认风险无影响，正常放行

线下处理     需要线下人工处理的风险


权限说明

操作权限：

管理员：可对所有风险记录进行处置、删除、批量操作等所有操作。

普通用户：仅可查看权限内的风险记录，进行处置操作，无法删除或批量操作。

数据权限：按角色控制风险分类的可见性，如人事部门用户仅可见人事风险记录，财务部门用户仅可见财务风险记录。


数据说明

1. 风险记录基础数据

字段名称     说明     约束 / 规则

风险名称     风险的具体名称     决策产生

风险分类     风险所属分类     从风险分类字典选择

实体 ID     关联的实体标识     唯一标识

风险等级     风险的严重程度     从风险等级字典选择

触发次数     风险被触发的次数     自动统计，只读

触发原因     风险触发的具体原因     文本描述，必填

入参     触发风险的相关参数     JSON 格式，记录触发时的参数

触发时间     风险首次触发的时间     系统自动生成，只读

更新时间     风险记录最后更新的时间     系统自动生成，只读

状态     风险当前状态     从风险状态字典选择，默认待处置

2. 处置记录数据

字段名称     说明     约束 / 规则

处置方式     处置风险的方式     从处置方式字典选择，必填

处置人     执行处置的用户     系统自动关联，必填

处置时间     处置执行的时间     系统自动生成，必填

处置说明     处置的详细说明     文本描述，选填


交互说明

查询

搜索框：

文本框输入风险名称进行模糊查询，按回车触发查询。

下拉框筛选：

筛选风险分类（人事风险、运营风险等）。

筛选风险等级（高风险、中风险、低风险）。

表格头筛选：

支持按【风险名称、风险分类、实体ID、风险等级、状态、触发时间（范围）、更新时间（范围）、处置方式、处置时间（范围）】筛选。

表格头排序：

支持按【风险分类、触发时间、更新时间、处置时间、风险等级、状态】排序。


风险处置交互

单条处置：

点击 “处置” 按钮，弹出处置窗口。

显示风险基础信息：风险名称、分类、实体 ID、触发次数、触发原因。

选择处置方式，输入处置说明（放行必填，人工处理不必填）。

点击 “提交” 按钮，前端验证处置方式必填。

提交后按钮显示加载状态，发送请求至后端。

成功后关闭窗口，更新风险状态为已处置；失败保持窗口，提示原因。

批量处置：

选中多条风险记录，点击 “批量处置” 按钮。

弹出批量处置窗口，显示选中的风险记录列表。

处置说明（放行必填，人工处理不必填）

点击 “提交” 按钮，前端验证处置方式必填。

提交后按钮显示加载状态，发送请求至后端。

成功后关闭窗口，更新选中风险状态；失败保持窗口，提示原因。


危险操作

删除风险：

点击 “删除” 按钮，弹出确认弹框，提示 “删除后无法恢复，是否继续？”。

确认后执行删除操作，仅管理员可操作。


提交交互

点击 “提交” 按钮后，前端验证所有必填字段（处置方式），通过后发送请求至后端，按钮显示加载状态，拿到结果后解除。成功关闭窗口，失败保持窗口，告知原因，方便用户修改。


其他交互

无特殊情况，按原型开发即可不做任何说明。
