应用中心需求说明

业务说明

目标场景和背景说明

1、为租户提供应用开通与续费的统一管理平台，支持租户根据业务需求自主订阅应用服务，实现应用使用的灵活性与成本可控性。通过该平台，租户可独立管理其订阅的应用，减少与系统核心功能的耦合，便于后续业务扩展与维护。

2、平台采用微前端架构，支持租户在订阅应用后，通过底层服务 API 调用相关功能，专注于自有业务。例如，租户订阅 “应收管理” 应用后，可直接使用该应用功能，而人员、组织架构等基础服务仍由系统管理服务提供。

3、系统应用（如 “系统配置”“组织架构” 等）为租户默认提供，无需开通与续费，确保租户基础业务流程的正常运行；常规应用（如 “应收管理”“账单管理” 等）由租户自主订阅，满足个性化业务需求。


功能说明

【应用中心】用于展示应用列表、支持租户开通新应用、对已订阅应用进行续费操作。


字典说明

1、应用类型：
  系统应用：默认提供给租户，无需开通与续费，价格一律为 0 元，状态不可变更。例如 “系统配置”“组织架构” 等。

  常规应用：租户可自主开通与续费的应用，支持所有订阅相关操作。例如 “应收管理”“账单管理” 等。

2、订阅状态：

已开通：租户已订阅且在有效期内的应用，可正常使用。

未开通：租户未订阅的常规应用，可进行开通操作。

将过期：已开通的应用距离有效期结束剩余天数≤30 天（可配置），需提醒租户续费。

已过期：已开通的应用超过有效期，自动停用，需租户续费后重新启用。


权限说明

操作权限：仅租户管理员可进行应用开通与续费操作，普通租户无此权限。

页面权限：租户登录后可查看应用中心页面，系统应用对所有租户可见，常规应用根据订阅状态显示。


数据说明

应用价格：常规应用显示实际订阅价格，系统应用价格为 0 元。

有效期：

    1、开通应用时默认设置有效期（如 1 年），可手动修改。

    2、有效期剩余天数 = 有效期结束日期 - 当前日期。

    3、当剩余天数≤30 天时，应用状态标记为 “将过期”；剩余天数≤0 天时，标记为 “已过期”。

订阅记录：记录租户的应用开通与续费历史，包括应用 ID、租户 ID、操作类型（开通 / 续费）、操作时间、支付金额等。

交互说明

查询

筛选下拉框：

    选项包括 “全部”“已开通”“未开通”“将过期”“已过期”。

   选择不同选项时自动刷新应用列表，减少用户点击操作。

搜索框：

   输入应用名称后按回车自动触发搜索，支持模糊查询。

   搜索结果实时显示在应用列表中。


排序

将要过期，已开通，未开通，系统应用的顺序展示


应用列表展示

系统应用：

显示应用名称、描述。

价格显示为 “¥0.00 元”，不显示续费按钮。

常规应用：

显示应用名称、描述、价格、有效期剩余天数。

根据订阅状态显示不同操作按钮：

未开通：显示 “开通” 按钮。

已开通,将过期：显示 “有效期剩余 XX 天” 和 “续费” 按钮，按钮样式为橙色提醒。

已过期：显示 “已过期” 和 “续费” 按钮，按钮样式为红色。


开通 / 续费操作

开通应用、续费操作，价格汇总。


其他交互

价格显示：常规应用显示具体价格（如 “¥800 元”），系统应用显示 “¥0.00 元”。

合计金额：在应用列表底部显示所有新增开通和续费常规应用的合计金额，方便租户查看总费用。


其他交互

无特殊情况，按原型开发即可不做任何说明。


数据埋点

1、用处

分析租户应用订阅行为，为应用推广、定价策略优化、租户服务提升提供数据支撑。

2、埋点数据

应用查看数：记录【应用 ID，租户 ID，查看时间】，形成日报，分析租户对不同应用的关注度。

开通操作数：记录【应用 ID，租户 ID，开通时间，支付金额】，形成日报，统计各应用的开通情况。

续费操作数：记录【应用 ID，租户 ID，续费时间，支付金额，原有效期剩余天数】，形成日报，分析租户续费趋势。

筛选点击数：记录【筛选条件，租户 ID，点击时间】，了解租户常用的筛选方式，优化筛选功能。

搜索关键词：记录【搜索关键词，租户 ID，搜索时间】，分析租户需求，优化应用名称与分类。
