数据源管理需求说明

业务说明


产品说明书

https://ycnfozkwomo9.feishu.cn/docx/I3JPd8MR9oUNqEx4Oj3c5f4Kn3g?from=from_copylink


目标场景和背景说明

1、为风控平台与数据分析平台提供统一的数据源管理能力，实现数据源的标准化接入与集中管理，减少各平台对数据源的重复配置，降低业务耦合度。

2、支持后续开发者无限扩展数据源类型与数量，开发者通过底层服务 API 调用数据源接口时，只需关注业务逻辑，无需重复处理数据源连接细节。例如，风控平台调用 MySQL 数据源进行风险数据查询，数据分析平台调用 ClickHouse 数据源进行大数据分析，均通过统一的数据源管理接口实现。

3、采用微服务架构理念，先实现前端数据源管理功能，为后续全链路微服务化奠定基础。通过前端先行，快速满足风控与数据分析平台的数据源管理需求，后续逐步完善底层微服务支持。


功能说明

【数据源管理】用于管理风控平台与数据分析平台所需数据源的创建、编辑、删除、查询、状态管理等操作。


字典说明

1、数据源分类：

MySQL     关系型数据库，适用于结构化数据存储与查询

PostgreSQL     强类型关系型数据库，支持复杂查询与扩展

Oracle     企业级关系型数据库，适用于大型业务系统

SQL Server     Microsoft 旗下关系型数据库，与 Windows 生态集成

达梦数据库     国产关系型数据库，支持国产化部署需求

MongoDB     文档型非关系数据库，适用于非结构化数据

Elasticsearch     分布式搜索与分析引擎，用于全文检索与数据分析

ClickHouse     列式存储数据库，适用于海量数据实时分析

Hive     基于 Hadoop 的数据仓库工具，支持大数据离线分析

2. 数据源状态:

启用：默认状态，数据源可被风控平台与数据分析平台正常调用。

停用：停用的数据源无法被平台调用，需启用后才可使用。


权限说明

操作权限：设置数据源的增删改查操作权限，不同角色（如管理员、开发人员）拥有不同权限。

页面权限：能进入数据源管理页面的用户，根据其角色权限执行相应操作，确保数据源配置的安全性。


数据说明

数据源名称：数据唯一，用于标识数据源，便于用户识别。

创建时间 / 更新时间：自动记录数据源的创建与更新时间，用于版本追溯。

创建人：记录创建数据源的用户信息，便于责任追溯。

交互说明

查询

筛选下拉框：

选项包括数据源分类（如 MySQL、PostgreSQL 等），选择不同分类时自动触发查询。

新增筛选条件（如状态：启用 / 停用），方便用户快速定位数据源。

搜索框：

输入数据源名称后按回车自动触发搜索，支持模糊查询。

搜索结果实时显示在数据源列表中。


排序

表头支持排序字段：【创建时间、更新时间、数据源名称、状态、数据源分类】，用户可点击表头按升序或降序排列。


表格更新

状态修改：

系统数据源（如基础服务数据源）无法修改状态，其余数据源可修改。

状态改变触发更新请求，请求过程中禁用开关，请求结束后解禁，修改失败则状态恢复修改前。

其他字段更新：暂不支持直接在表格中修改其他字段，需进入编辑页面操作。


新增 / 编辑数据源

必填字段校验：

数据源名称、数据源分类为必填项，未填写时表单无法提交。

port 需为数字，范围合理（如 1-65535），否则提示错误。

默认值填充：

状态默认选中 “启用”。

数据源分类下拉框：

显示所有支持的数据源分类（如 MySQL、MongoDB 等），用户根据实际数据源类型选择。

密码输入：

密码输入框需加密显示（如星号），保障数据安全。

提交与验证：

点击 “提交” 按钮后，前端验证所有字段格式，通过后发送请求至后端。

后端接收到请求后，尝试连接数据源，验证连接是否成功。

连接成功则保存数据源配置，失败则返回错误原因（如用户名密码错误、端口不可访问等），提示用户修改。


删除操作

危险操作确认：删除数据源时弹出确认弹框，提示 “删除后将无法恢复，且可能影响相关平台使用，是否继续？”，避免误操作。

关联检查：删除前检查数据源是否被风控平台或数据分析平台使用，若有引用则提示无法删除，需先解除引用关系。


其他交互

无特殊情况，按原型开发即可不做任何说明。
